Design Highlights

  • Route responsibilities are separated by domain-focused blueprints.
  • Task APIs expose JSON payloads for dashboard analytics and extensions.
  • Security headers are appended globally at response level.
  • Data model is intentionally minimal for easy migration to production RDBMS.

    Browser->>Flask: HTTP request
    Flask->>Blueprint: Route dispatch
    Blueprint->>DB: Query or mutate entities
    DB-->>Blueprint: Persisted state
    Blueprint-->>Flask: Template or JSON payload
    Flask-->>Browser: Response + security headers
